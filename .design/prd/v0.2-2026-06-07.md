# PRD: 2026 世界杯球迷互动站

## 版本
v0.2 - 2026-06-07

## 概述
面向中国球迷的 2026 FIFA 世界杯娱乐互动网站，提供赛程比分、竞猜、球星卡收集、排行、社区讨论等功能，以提升用户参与度和粘性为核心目标，非盈利性质。

## 目标用户
- 中文世界杯球迷（18-40岁）
- 有竞猜/互动需求的活跃球迷
- 边看球边刷手机聊球的群体

## 技术选型
- **技术栈**: Next.js 14 (App Router) + TypeScript + Tailwind CSS + Shadcn/ui
- **数据库**: PostgreSQL + Prisma ORM
- **部署**: 阿里云 ECS + Docker（待定）
- **包管理**: pnpm
- **版本控制**: Git

## 功能详情

### 1. 赛程 & 实时比分

**数据来源：**
- 世界杯官方赛程数据：先期手动录入 2026 年全部 104 场比赛信息
- 实时比分：比赛中通过手动更新或模拟数据更新
- MVP 阶段暂不接入外部 API，避免依赖不稳定

**展示规则：**
- 首页默认展示「今日比赛」，按时间排列
- 未开始的比赛显示 ⏳ 状态，比分显示 `-` `-`
- 进行中的比赛显示 🔴 LIVE 标签，比分实时更新
- 已结束的比赛显示 ✅ FT，比分固定

**每场比赛卡片展示：**
- 主队国旗、主队名 vs 客队国旗、客队名
- 比分（实时或最终）
- 比赛时间、分组
- 状态标签（未开始 / LIVE / FT）
- 「预测」按钮（未开始的比赛才可点）

**交互：**
- 点击比赛卡片可展开查看详情
- 支持按日期切换查看赛程

**更新频率：**
- 无 WebSocket，采用轮询方式，每 30 秒拉取一次最新比分

### 2. 每日竞猜

**竞猜范围：**
- 用户可预测**当日所有未开始**的比赛
- 一场比赛只能提交一次预测

**竞猜内容（两种模式组合）：**
- 用户需要同时预测：**胜负关系 + 具体比分**
  - 胜负预测：主队胜 / 客队胜 / 平局（仅小组赛有平局，淘汰赛直接预测 90分钟胜平负）
  - 比分预测：主队进球数、客队进球数（0-9）

**积分规则：**

| 命中项 | 得分 | 说明 |
|--------|:---:|------|
| 胜负预测正确 | +10 分 | 只看胜/平/负 |
| 比分完全正确 | +30 分 | 含胜负分，不叠加 |
| 比分差正确（如2-1猜中净胜1球） | +15 分 | 含胜负分，不与上述叠加 |
| 猜错 | 0 分 | 不扣分 |

**连胜加成（Streak）：**

| 连续猜中次数 | 额外加成 |
|:----------:|:--------:|
| 1~2 场 | 无加成 |
| 3~4 场 | 每场 +3 分 |
| 5~7 场 | 每场 +5 分 |
| 8+ 场 | 每场 +10 分 |

- 连胜只计算「胜负预测正确」
- 猜错一场连胜中断，重新计数
- 每日竞猜结束后更新积分

**截止时间：**
- 每场比赛**开赛前 5 分钟**截止预测
- 截止后预测按钮灰色不可点

**修改规则：**
- 截止时间前**可以修改**预测，以最后一次为准
- 截止后不可修改

**结果显示：**
- 比赛结束后，用户可查看自己的预测结果
- 显示：你的预测 vs 实际结果 → 命中/未命中 → 获得积分
- 每场竞猜结果用绿色（命中）/ 红色（未中）标注

### 3. 积分排行榜

**排名维度：**

| 排行类型 | 周期 | 更新频率 | 说明 |
|---------|:---:|:--------:|------|
| 日榜 | 当天 | 实时 | 每日竞猜结算后更新 |
| 周榜 | 本周 | 实时 | 周一 00:00 重置 |
| 总榜 | 全部 | 实时 | 世界杯全程累计 |

**显示内容：**
- 排名（🥇🥇🥇 前三特殊样式）
- 用户头像（默认生成彩色首字母头像）
- 用户名
- 总积分
- 连胜场次（用 🔥 火焰图标表示）
- 胜率（猜对场次 / 总场次）

**排名规则：**
- 按总积分从高到低排序
- 积分相同则按胜率排名
- 胜率相同则按先达到该积分的排名靠前

**头衔系统（当积分达到自动获得）：**

| 积分门槛 | 头衔 |
|:------:|------|
| 0 | 菜鸟球迷 |
| 50 | 初级球探 |
| 150 | 资深球迷 |
| 300 | 足球分析师 |
| 500 | 预测大师 |
| 800 | 世界杯百晓生 |
| 1200 | 传奇预言家 |

**UI 展示：**
- 首页只展示 Top 5
- 完整排行榜在独立页面展示所有用户

### 4. 球星卡收集

**卡牌种类：**
- 共 **100 张** 世界杯球员卡
- 覆盖 32 支热门球队的核心球员
- 每张卡包含：球员名、国家、国旗、稀有度、球员位置

**稀有度与概率：**

| 稀有度 | 概率 | 卡牌数量 | 标识 |
|:----:|:---:|:--------:|------|
| ⚪ 普通 (Common) | 60% | 60 张 | 灰白边框 |
| 🔵 稀有 (Rare) | 30% | 30 张 | 蓝光边框 |
| 🟡 传奇 (Legendary) | 10% | 10 张 | 金色闪烁边框 |

**卡包获取方式：**

| 方式 | 次数 | 说明 |
|------|:---:|------|
| 每日免费 | 1 包/天 | 登录即送，每日 00:00 重置 |
| 竞猜奖励 | 猜中 2 场送 1 包 | 每日上限 3 包 |
| 签到奖励 | P1 阶段 | 暂未开放 |

**开包规则：**
- 每包包含 **3 张** 随机球员卡
- 按概率池随机抽取
- 开包时展示开启动效（卡牌翻转效果）
- 已拥有的卡会标注「已收集」

**重复卡处理：**
- 重复卡自动转化为 **碎片**
- 普通卡 → 10 碎片
- 稀有卡 → 30 碎片
- 传奇卡 → 100 碎片

**碎片用途（P1 阶段）：**
- 100 碎片可合成一张指定普通卡
- 300 碎片可合成一张指定稀有卡
- 碎片系统作为 P1 迭代

**收集进度：**
- 首页显示 **进度条 + 百分比**
- 显示：已收集 X / 100 张
- 按稀有度分类显示收集情况

**图鉴页面（独立页）：**
- 全部 100 张卡展示
- 已收集：彩色显示
- 未收集：灰色剪影 + ❓
- 可按国家/稀有度筛选

### 5. 社区讨论区

**用户身份：**
- MVP 阶段**无需注册登录**
- 首次进入时让用户输入一个**昵称**（2-8个中文字符或英文）
- 昵称存在浏览器 localStorage 中
- 用户标识用设备生成唯一 ID

**发帖规则：**
- 内容：纯文本 + emoji，最多 200 字
- 每天最多发 5 条
- 禁止：广告、人身攻击、政治敏感内容
- MVP 阶段不设审核，先发后审（看情况）

**帖子展示：**
- 首页显示最新 2 条帖子
- 社区独立页按**时间倒序**展示所有帖子
- 每条帖子显示：用户头像（昵称首字母）、昵称、时间、内容
- 帖子按「X分钟前」格式显示相对时间

**互动功能：**

| 功能 | 规则 |
|------|------|
| 👍 点赞 | 每人每帖只能点一次，可取消 |
| 💬 评论 | 直接回复，不限层级（统一平铺） |
| 举报 | 暂不开放 |

**帖子排序：**
- 默认：按发布时间倒序
- 可切换：按热度排序（点赞数 + 评论数 * 2）

**发帖入口：**
- 首页底部悬浮「+」按钮
- 社区页面顶部输入框

**数据存储：**
- MVP 阶段帖子数据存数据库
- 昵称 + 设备 ID 关联用户

### 6. 用户体系（MVP 简化版）

| 项 | 方案 |
|------|------|
| **登录** | 无需登录 |
| **用户标识** | 浏览器生成设备 UUID，存 localStorage |
| **昵称** | 首次进入让用户自定义，存 localStorage |
| **数据存储** | 关联设备 UUID 存到数据库 |
| **风险** | 清浏览器或换设备数据丢失 |

## 非功能性需求
- 移动端优先适配（手机浏览器占 80%+ 流量）
- 首屏加载 < 3 秒
- 配色统一，使用绿茵主题色
- 无登录系统，降低使用门槛
- 页面切换无刷新（SPA 体验）

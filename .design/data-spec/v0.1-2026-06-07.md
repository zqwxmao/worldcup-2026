## 二、FIFA 官方 API（首选方案）

### 接口地址
FIFA 使用以下基础域名对内/对外提供 JSON 数据：
```
https://api.fifa.com/api/v3/
```

### 已知有用接口

| 接口 | 说明 |
|------|------|
| `/calendar/matches?idSeason={seasonId}&language=zh-Hans` | 获取赛季全部比赛赛程 |
| `/live/football` | 获取实时比赛数据和比分 |
| `/teams/{teamId}` | 获取球队详细信息 |

> **注意：** FIFA API 尚未发布 2026 世界杯的正式 seasonId，通常在开赛前 6 个月左右上线。
> 上线前可先用 API-FOOTBALL 或模拟数据替代。

### 请求方式
```
GET https://api.fifa.com/api/v3/calendar/matches
  ?idSeason=2026
  &language=zh-Hans
```
- 无需 API Key
- 无需注册
- 返回 JSON 格式
- 支持中文参数（`zh-Hans`）

### 国内可访问性
✅ FIFA API 国内可直接访问，不翻墙

### 限流
- 未公开限制，但建议控制频率
- 建议缓存策略：每次请求结果存数据库，5 分钟内不重复请求

## 三、备选方案：API-FOOTBALL（RapidAPI）

### 基本信息
- **网址**：https://rapidapi.com/api-sports/api/api-football/
- **注册**：邮箱注册 RapidAPI 账号
- **免费额度**：**100 次/天**
- **数据覆盖**：赛程、实时比分、球队、球员、统计等

### 国内可访问性
⚠️ RapidAPI 在国内部分地区可能被墙，需要 VPN/代理

### 使用时机
- FIFA API 未上线或不可用时作为替代
- MVP 初期 100 次/天足够（每 30 秒轮询比分，一天约 2880 次 → 需要缓存策略控制）

### 缓存策略（配合免费额度）
```
第一层：内存缓存（Node.js 变量） → 有效期 15 秒
第二层：数据库缓存              → 有效期 5 分钟
第三层：API 调用               → 仅在缓存过期时触发
```
通过缓存，100 次/天的免费额度完全够用。

## 四、兜底方案：爬虫

### 爬取目标
如果以上 API 都不可用，从以下站点爬取数据：
1. **ESPN 世界杯专区**：`https://www.espn.com/soccer/scoreboard/_/league/fifa.world/`
2. **腾讯体育**：`https://matchweb.sports.qq.com/`（有 JSONP 接口）
3. **FIFA 官网页面**：`https://www.fifa.com/en/tournaments/mens/worldcup/`

### 爬虫策略
- 使用 `node-fetch` 或 `axios` 直接抓取
- 解析 HTML 或隐藏的 JSON 数据
- 定时任务（cron）每 5 分钟抓一次，写入数据库
- MVP 阶段暂不实现爬虫，仅作为备案

## 五、数据缓存方案

### 架构
```
用户请求 → 内存缓存（15秒）→ 数据库缓存（5分钟）→ API/爬虫
```

### 各层说明

| 层级 | 存储位置 | 有效期 | 作用 |
|------|---------|:-----:|------|
| L1 | Node.js 内存变量 | 15 秒 | 极速响应，减少重复请求 |
| L2 | PostgreSQL 数据库 | 5 分钟 | 持久化，跨请求共享 |
| L3 | 外部 API | 按需 | 源数据获取 |

### 定时任务（Cron）

| 任务 | 频率 | 说明 |
|------|:---:|------|
| 拉取今日赛程 | 每天 00:05 | 获取当天所有比赛 |
| 更新实时比分 | 每 30 秒 | 仅在比赛进行时段（有 LIVE 比赛时） |
| 清理过期缓存 | 每天 03:00 | 删除超过 1 天的缓存数据 |

### 实现方式
- Next.js 内置 `setInterval` 或 `cron-job.org` 外部定时器
- 或使用 Vercel Cron Jobs / 阿里云定时触发器（部署后决定）

## 六、静态数据（手动录入一次）

以下数据只需录入一次，后续基本不变：

| 数据 | 内容 | 说明 |
|------|------|------|
| 球星卡 100 张 | 球员名、国家、国旗、稀有度、位置 | 手动构建一个 JSON 数组 |
| 球队基本信息 | 队名、国旗 emoji、分组 | 从维基百科整理 |
| 积分头衔 | 7 个等级名称和门槛 | PRD v0.2 已定义 |

这些数据量不大（100 条卡牌、32 队），直接写在代码的 `data/` 目录下作为静态数据源，不存数据库（减少复杂度）。

## 七、错误处理与降级

| 场景 | 处理方式 |
|------|---------|
| API 请求失败 | 返回上次缓存的旧数据 |
| 连续失败 3 次 | 暂停该 API 请求 10 分钟，避免被封 |
| 无可用数据 | 展示「数据加载中…」占位 |
| 比赛开始但无比分 | 显示 ⏳ 等待中 |

## 项目初始化步骤

### 1. 创建项目

```bash
# 进入项目目录
cd /home/ubuntu/worldcup-website

# 使用 pnpm 创建 Next.js 项目
pnpm create next-app@latest . --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"

# 安装核心依赖
pnpm add @prisma/client
pnpm add -D prisma

# 安装 UI 组件库
pnpm add @radix-ui/react-dialog @radix-ui/react-dropdown-menu @radix-ui/react-tabs
pnpm add class-variance-authority clsx tailwind-merge lucide-react
pnpm add @tanstack/react-query  # 数据请求管理

# 安装 Shadcn/ui 组件库（按需添加）
npx shadcn-ui@latest init
npx shadcn-ui@latest add button card dialog tabs dropdown-menu
```

### 2. 初始化 Git

```bash
cd /home/ubuntu/worldcup-website
git init
git add .
git commit -m "🎉 init: 项目初始化"

# 关联远程仓库（按实际情况修改）
# git remote add origin git@github.com:yourname/worldcup-website.git
# git push -u origin main
```

### 3. 初始化数据库

```bash
# 初始化 Prisma
npx prisma init

# 创建初始数据库模型后执行迁移
npx prisma migrate dev --name init
```

### 4. 开发环境启动

```bash
# 启动开发服务器（默认 http://localhost:3000）
pnpm dev
```

## 数据库模型设计（初版）

```prisma
// prisma/schema.prisma

model Match {
  id          String   @id @default(cuid())
  homeTeam    String   // 主队名
  awayTeam    String   // 客队名
  homeFlag    String   // 主队国旗 emoji
  awayFlag    String   // 客队国旗 emoji
  group       String   // 分组
  matchTime   DateTime // 比赛时间
  homeScore   Int?     // 主队比分（null 表示未开始）
  awayScore   Int?     // 客队比分
  status      String   // scheduled / live / finished
  createdAt   DateTime @default(now())
}

model Prediction {
  id        String   @id @default(cuid())
  matchId   String
  userId    String   // 用户标识（MVP 阶段可用设备ID或简单用户名）
  predictHomeScore Int
  predictAwayScore Int
  points    Int?     // 猜对得分
  createdAt DateTime @default(now())

  match     Match    @relation(fields: [matchId], references: [id])
}

model PlayerCard {
  id          String   @id @default(cuid())
  name        String   // 球员名
  country     String   // 国家
  countryFlag String   // 国旗 emoji
  rarity      String   // common / rare / legendary
  imageUrl    String?  // 卡牌图片 URL
  createdAt   DateTime @default(now())
}

model UserCard {
  id        String   @id @default(cuid())
  userId    String
  cardId    String
  obtainedAt DateTime @default(now())

  card      PlayerCard @relation(fields: [cardId], references: [id])
}

model User {
  id        String   @id @default(cuid())
  name      String   // 用户名
  points    Int      @default(0)   // 竞猜积分
  streak    Int      @default(0)   // 连胜次数
  createdAt DateTime @default(now())
}

model Post {
  id        String   @id @default(cuid())
  userId    String
  userName  String
  content   String
  likes     Int      @default(0)
  createdAt DateTime @default(now())
}

model Comment {
  id        String   @id @default(cuid())
  postId    String
  userId    String
  content   String
  createdAt DateTime @default(now())

  post      Post     @relation(fields: [postId], references: [id])
}
```

## API 接口设计（初版）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/matches | 获取赛程列表 |

## 开发流程

### 分支管理

```
main        ← 稳定版本，可部署
├── dev     ← 开发分支
├── feat/*  ← 功能分支（feat/predict, feat/cards）

流程:
main ← dev ← feat/*
```

### Commit 规范

```
feat: 新功能
fix: 修复 bug
docs: 文档更新
style: 样式修改
refactor: 重构
chore: 构建/工具
```

示例：
```
feat: 添加赛程列表组件
fix: 修复排行榜分数计算错误
docs: 更新 PRD 到 v0.2
```

### 开发顺序（按 MVP 优先级）

1. **Phase 1** — 项目初始化 + 数据库搭建 + 赛程页面
2. **Phase 2** — 竞猜功能 + 排行榜
3. **Phase 3** — 球星卡收集系统
4. **Phase 4** — 社区讨论区
